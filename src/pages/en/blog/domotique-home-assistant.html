My smart home with Home Assistant

March 2026 • Experience report — Personal project

The starting point

The idea was simple: control the entire house from a single screen. Before, each connected device had its own app, its own notifications, and no communication between them. I wanted a system that was unified, local and customizable.

I chose Home Assistant: open source, self-hosted, with an active community and very broad hardware compatibility.

The setup

1. The server

Home Assistant runs on a dedicated home server. The setup required configuring the local network, assigning a static IP, and setting up dashboard access via the browser.

2. The devices

I integrated about ten devices via a Tapo hub:

  • Door/window sensors: open/closed detection
  • Temperature sensors: room-by-room monitoring
  • Motion sensors: presence detection
  • Smart plugs and lights: remote control

3. The dashboard

A centralized dashboard displayed permanently on a dedicated tablet. It shows the status of all sensors, device controls, and active alerts. The interface is customized with cards tailored to each room in the house.

Automations

The heart of the project: YAML automations. I wrote about ten of them, each triggered by a sensor event:

  • Intrusion alert: notification if a door sensor triggers while in away mode
  • Lighting control: automatic on when motion detected, off after a delay
  • Temperature monitoring: alert if a room exceeds a configured threshold
  • Scenarios: "Good night" (all lights off, alarm on), "Leaving" (check doors, away mode)

Technical challenges

Hardware compatibility

Not all devices integrate as easily. Some sensors require specific integrations or manual configuration. The Tapo hub simplified communication, but each device had to be tested individually for reliability.

Automation reliability

A poorly calibrated automation can quickly become annoying: false positives on motion sensors, unwanted alerts, etc. Each automation was tested and adjusted over several days before being validated.

The house is now controlled from a single screen. Automations run 24/7 without intervention, and alerts arrive directly on my phone.

Key takeaways

  • Local first: self-hosting means keeping control of your data without depending on the cloud.
  • YAML is powerful: text-based automations are versionable, readable, and editable without a GUI.
  • Test before activating: each automation needs several days of testing to avoid false positives.
  • A good dashboard changes everything: a centralized tablet interface makes the system usable by the whole family.
