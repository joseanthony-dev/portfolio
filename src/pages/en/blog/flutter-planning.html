Building a team planning app with Flutter and Firebase

March 2026 • Experience report — Personal project

FlutterFirebaseFirestorePDF

The need

In my order preparation team, we meet every Saturday for events. The problem: nobody knew who was coming, at what time, or how many hours each person had worked. Everything went through group messages, with no history or reliable tracking.

I decided to build a dedicated app to centralize all of this: events, attendance, hours and communication.

The tech stack

Why Flutter?

I wanted a native mobile app, fast to develop, that runs on Android and iOS with a single codebase. Flutter checked all the boxes: reactive UI, hot reload for fast iteration, and a solid package ecosystem.

Why Firebase?

For the backend, Firebase was the obvious choice: Firestore for real-time data (attendance responses update instantly), built-in authentication, and zero servers to manage.

Features

  • Event creation: date, location, description — the organizer publishes, the team responds
  • Attendance tracking: each member indicates if they're coming or not, visible to all in real time
  • Hour tracking: each participant logs their hours, the admin sees the total
  • PDF export: generates a clean summary with attendance and hours per event
  • SMS and email notifications: automatic reminders before each event and direct contact

Technical challenges

PDF generation

Generating a clean PDF from Flutter is not trivial. I had to build the layout manually (tables, alignment, styles) with the pdf package, then handle file sharing across platforms (Android/iOS).

Real-time with Firestore

Real-time is a major asset, but data needs to be well structured. I had to rework the Firestore model several times to avoid excessive reads (and associated costs) while keeping reactivity.

Multi-channel notifications

Integrating SMS and email in a Flutter app requires external services. The challenge was making it reliable and handling error cases (invalid number, undelivered email) without blocking the user experience.

The app is functional and covers the entire initial need: event creation, attendance management, hour tracking with PDF export, and integrated notifications. Production rollout to the team is planned soon.

Key takeaways

  • Start from a real need: the app solves a concrete problem for my team, not a theoretical exercise.
  • Flutter accelerates development: hot reload and the widget system allow very fast UI iteration.
  • Firebase simplifies the backend: no server to manage, real-time is native, and auth is ready to use.
  • PDF is more work than expected: generating a clean document from mobile takes more effort than you think.
